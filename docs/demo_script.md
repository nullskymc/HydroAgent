# HydroAgent 答辩演示脚本

本文档用于毕业设计现场演示。建议演示前先确认后端、前端和配置文件可用，避免现场临时调试。

## 演示准备

1. 启动系统：

   ```bash
   ./start.sh
   ```

2. 打开前端：

   ```text
   http://127.0.0.1:3000
   ```

3. 使用管理员账号登录：

   ```text
   admin / admin123
   ```

4. 如果需要单独验证后端健康状态：

   ```text
   http://127.0.0.1:7860/api/health
   ```

## 演示主线

### 1. 登录与权限

讲解点：

- 系统有管理员、运营经理、操作员、观察员、审计员等角色。
- 前端页面根据后端 RBAC 权限显示和跳转。
- 登录后可以看到当前用户、角色和可访问模块。

演示操作：

- 进入登录页。
- 使用 `admin / admin123` 登录。
- 指出顶部导航中的运营总览、智能对话、运营中心、资产中心、知识库、告警中心、用户与角色、报表中心、审计记录和系统设置。

### 2. 运营总览

讲解点：

- 首页展示系统在线状态、平均湿度、待审批计划、天气和分区状态。
- 分区表把传感器数据、阈值、湿度缺口和执行器状态聚合到同一个视图。
- 手动启动仍会遵守“先生成计划、再审批、再执行”的安全边界。

演示操作：

- 打开“运营总览”。
- 展示运行总览、分区状态、天气预报和策略边界。
- 说明传感器数据在毕业设计环境中使用仿真数据。

### 3. 生成灌溉计划

讲解点：

- 系统按分区生成结构化计划。
- 计划包含证据、风险等级、安全审查、建议动作和建议时长。
- 未来 48 小时有降雨或传感器异常时，系统会降低自动执行倾向。

演示操作：

- 进入“运营中心”或通过“智能对话”请求生成计划。
- 展示生成后的计划状态，例如 `pending`、`approved`、`executed` 或 `ready`。
- 说明 `start` 类计划需要审批。

可用话术：

```text
请分析当前分区湿度，并为需要灌溉的分区生成灌溉计划。
```

### 4. 审批与执行

讲解点：

- 批准计划会生成审批记录。
- 执行计划会更新计划状态、执行器状态和灌溉日志。
- 拒绝计划不可执行，保证安全边界。

演示操作：

- 在“运营中心”选择待审批计划。
- 点击“批准”。
- 点击“执行”。
- 展示最近执行日志。

### 5. 告警与审计

讲解点：

- 告警中心用于处理湿度异常、传感器异常和执行器风险。
- 审计记录保存登录、计划、设置和管理操作。
- 报表中心可以导出运营、审计和分区报表。

演示操作：

- 打开“告警中心”，展示告警状态和处理动作。
- 打开“审计记录”，展示计划生成、审批、执行或登录事件。
- 打开“报表中心”，展示导出入口。

### 6. 知识库与智能对话

讲解点：

- 知识库可保存设备手册、运维 SOP、业务规则等文本。
- 智能体可以结合工具调用和系统数据回答灌溉问题。
- 对话轨迹和工具调用记录可用于审计和解释。

演示操作：

- 打开“知识库”，新增一段简短 SOP。
- 打开“智能对话”，询问当前灌溉建议。
- 展示响应中的计划、工具轨迹或决策记录。

可用话术：

```text
当前是否适合启动灌溉？请说明依据、风险和下一步操作。
```

## 答辩强调点

- 本项目不是简单页面展示，而是有后端业务模型、服务层、安全规则、审批链路和审计记录的完整原型。
- “仿真”不是未完成，而是毕业设计环境中对真实硬件的可控替代；真实接入只需要替换传感器和执行器适配层。
- 系统重点体现了智能灌溉的闭环：感知数据、天气风险、计划生成、人工审批、安全执行、日志审计、可视化运维。
- 前端已通过 TypeScript、ESLint 和生产构建检查。
- 旧测试仍需按新架构重建，最终验收应以当前 FastAPI 服务层和 Next.js 控制台为主。

## 常见问题回答

**为什么没有接真实水泵？**

毕业设计环境下真实水泵涉及硬件安全、场地和接口协议。系统已把执行器抽象为 `Actuator` 和计划执行服务，后续可替换为 MQTT、Modbus、HTTP 网关或串口适配器。

**AI 会不会直接乱启动灌溉？**

不会。系统规则要求 `start` 必须基于结构化计划，并经过审批；执行前还会检查计划状态、审批状态、执行器可用性和风险条件。

**天气接口失败怎么办？**

天气模块失败时会返回兜底摘要，保证系统能继续生成保守计划。真实生产环境可以增加缓存和多天气源降级。

**ML 模型是否已经训练？**

当前保留了 `SoilMoisturePredictor` 接口和预测流程，但默认是 DummyModel。论文中应表述为“预留机器学习预测扩展点”，不要写成已完成真实模型训练。
